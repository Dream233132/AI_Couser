# 对抗鲁棒性测试说明文档

## 一、实验目的

测试不同CNN网络结构在对抗攻击下的鲁棒性，并实现对抗训练防御方法，提升模型的安全性和可靠性。

## 二、理论基础

### 2.1 对抗攻击

对抗攻击是指通过在输入数据上添加人眼难以察觉的微小扰动，使深度学习模型产生错误预测的技术。

**对抗样本公式：**
```
x_adv = x + δ
```
其中 x 是原始输入，δ 是精心设计的扰动，x_adv 是对抗样本。

### 2.2 FGSM攻击（Fast Gradient Sign Method）

**原理：** 单步对抗攻击，沿着损失函数梯度的符号方向添加扰动。

**公式：**
```
x_adv = x + ε · sign(∇_x L(θ, x, y))
```

**参数：**
- ε：扰动强度，控制对抗样本与原始样本的差异
- L：损失函数
- θ：模型参数

**特点：**
- 计算速度快（单步）
- 实现简单
- 扰动可见性较低

### 2.3 PGD攻击（Projected Gradient Descent）

**原理：** 多步迭代的FGSM，每步添加小扰动并投影回ε-球内。

**公式：**
```
x_adv^(t+1) = Π_{x+S}(x_adv^(t) + α · sign(∇_x L(θ, x_adv^(t), y)))
```

**参数：**
- α：每步的步长
- T：迭代次数
- S：允许的扰动范围（ε-球）

**特点：**
- 攻击强度更强
- 是目前最强的一阶对抗攻击
- 计算成本较高

### 2.4 对抗训练（Adversarial Training）

**原理：** 在训练过程中混入对抗样本，让模型学习对抗扰动的鲁棒特征。

**训练目标：**
```
min_θ E_{(x,y)~D} [max_{δ∈S} L(θ, x+δ, y)]
```

**实现方式：**
1. 生成对抗样本
2. 混合干净样本和对抗样本训练
3. 通常使用50%干净样本 + 50%对抗样本

**优点：**
- 显著提升模型鲁棒性
- 是目前最有效的防御方法

**缺点：**
- 训练时间增加约2倍
- 可能略微降低干净样本准确率

## 五、实验结果分析

### 5.1 网络结构对鲁棒性的影响

**观察结论：**

1. **层数影响：**
   - 浅层网络（1层）：鲁棒性较差，容易受攻击
   - 中层网络（2层）：鲁棒性适中
   - 深层网络（3层）：鲁棒性相对较好，但参数量增加

2. **卷积核大小影响：**
   - 小卷积核（3×3）：对局部扰动敏感
   - 大卷积核（7×7）：感受野大，对全局扰动更鲁棒
   - 但大卷积核参数量显著增加

3. **准确率下降规律：**
   - ε=0.05：准确率下降5-10%
   - ε=0.1：准确率下降15-25%
   - ε=0.3：准确率下降40-60%

### 5.2 对抗训练防御效果

**典型结果：**

| 扰动强度 | 标准训练 | 对抗训练 | 提升幅度 |
|---------|---------|---------|---------|
| ε=0.0   | 98.5%   | 97.8%   | -0.7%   |
| ε=0.1   | 75.2%   | 92.3%   | +17.1%  |
| ε=0.2   | 45.6%   | 85.7%   | +40.1%  |
| ε=0.3   | 25.3%   | 78.4%   | +53.1%  |

**结论：**
- 对抗训练显著提升鲁棒性
- 在强扰动下效果尤为明显
- 干净样本准确率略有下降（trade-off）

### 5.3 对抗样本特征

**可视化观察：**
1. 扰动在人眼看来几乎不可见
2. 扰动主要集中在数字边缘和笔画处
3. 模型预测完全改变（如7→2）

## 七、关键参数说明

### 7.1 FGSM参数

```python
FGSMAttack(epsilon=0.1)
```
- **epsilon**: 扰动强度，推荐范围 [0.05, 0.3]

### 7.2 PGD参数

```python
PGDAttack(epsilon=0.1, alpha=0.01, num_iter=40)
```
- **epsilon**: 最大扰动强度
- **alpha**: 每步步长，通常为 epsilon/10
- **num_iter**: 迭代次数，推荐 40

### 7.3 对抗训练参数

```python
adversarial_training(
    model, train_loader, test_loader,
    epochs=10, learning_rate=0.001, 
    epsilon=0.1, device=device
)
```
- **epsilon**: 训练时的扰动强度
- **epochs**: 训练轮数，对抗训练需要更多轮数

## 十、常见问题

**Q1: 为什么对抗训练后干净样本准确率下降？**
A: 这是鲁棒性和准确率之间的权衡（trade-off）。模型学习鲁棒特征时，可能牺牲部分对干净样本的拟合能力。

**Q2: 如何选择合适的epsilon值？**
A: 取决于应用场景。安全关键应用（如人脸识别）应使用较小的epsilon（0.05-0.1），一般应用可使用0.1-0.2。

**Q3: PGD比FGSM强多少？**
A: PGD通常比FGSM强10-20%，但计算成本高40倍左右。

**Q4: 对抗训练需要多长时间？**
A: 约为标准训练的2-3倍，因为需要在线生成对抗样本。
