# 轻量化与移动端部署说明文档

## 一、实验目的

设计参数量<50k的超轻量CNN模型，在MNIST上达到>98%准确率，并将模型转换为ONNX格式，实现Web端实时手写数字识别演示。

## 二、轻量化模型设计

### 2.1 UltraLightCNN架构

**设计目标：**
- 参数量 < 50,000
- 准确率 > 98%
- 推理速度快

**网络结构：**
```
Block1: Conv(1→12) → Conv(12→12) → MaxPool  [28×28 → 14×14]
Block2: Conv(12→24) → Conv(24→24) → MaxPool [14×14 → 7×7]
Block3: Conv(24→40) → Conv(40→40)           [7×7]
Classifier: GAP → Dropout → FC(40→10)
```

**参数统计：**
- Block1: 1,452 参数
- Block2: 7,872 参数
- Block3: 23,200 参数
- Classifier: 410 参数
- **总计: 32,934 参数** ✅

### 2.2 轻量化技术

1. **减少通道数：** 12→24→40（而非32→64→128）
2. **全局平均池化：** 替代大型全连接层
3. **BatchNorm + Dropout：** 防止过拟合
4. **无bias卷积：** 配合BatchNorm减少参数

## 四、ONNX模型转换

### 4.1 转换代码

```python
import torch
import torch.onnx

# 加载模型
model = UltraLightCNN()
model.load_state_dict(torch.load('lightweight_cnn.pth'))
model.eval()

# 导出ONNX
dummy_input = torch.randn(1, 1, 28, 28)
torch.onnx.export(
    model, dummy_input,
    "lightweight_cnn.onnx",
    export_params=True,
    opset_version=11,
    input_names=['input'],
    output_names=['output']
)
```

### 4.2 ONNX优势

1. **跨平台：** 支持多种推理引擎
2. **高效：** 优化的计算图
3. **兼容性：** 易于部署到移动端/Web端

## 七、移动端部署建议

### 7.1 Android部署

使用ONNX Runtime Mobile：
```java
// 加载ONNX模型
OrtEnvironment env = OrtEnvironment.getEnvironment();
OrtSession session = env.createSession("lightweight_cnn.onnx");

// 推理
float[] input = preprocessImage(bitmap);
OnnxTensor tensor = OnnxTensor.createTensor(env, input);
OrtSession.Result result = session.run(Collections.singletonMap("input", tensor));
```

### 7.2 iOS部署

使用Core ML：
1. 转换ONNX → Core ML
2. 集成到Xcode项目
3. 使用Vision框架推理

### 7.3 Web部署

使用ONNX.js或TensorFlow.js：
```javascript
// 加载模型
const session = await ort.InferenceSession.create('lightweight_cnn.onnx');

// 推理
const tensor = new ort.Tensor('float32', inputData, [1, 1, 28, 28]);
const results = await session.run({ input: tensor });
```

## 八、优化技巧

### 8.1 进一步压缩

1. **量化：** INT8量化减少75%大小
2. **剪枝：** 移除不重要的连接
3. **知识蒸馏：** 用大模型指导小模型

### 8.2 加速推理

1. **批处理：** 多个样本一起推理
2. **模型融合：** 合并BN层到卷积层
3. **算子优化：** 使用优化的卷积实现

## 十、常见问题

**Q1: 为什么轻量化模型准确率反而更高？**
A: 合理的架构设计比单纯增加参数更重要。UltraLightCNN针对MNIST优化，避免了过拟合。

**Q2: ONNX模型能在哪些平台运行？**
A: Windows、Linux、macOS、Android、iOS、Web浏览器等几乎所有平台。

**Q3: Web端推理速度如何？**
A: 在现代浏览器中，单次推理约5-10ms，完全满足实时交互需求。

**Q4: 如何进一步减小模型？**
A: 可以使用INT8量化，将模型压缩到约30KB，准确率损失<0.5%。
